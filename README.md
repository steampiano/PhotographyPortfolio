# Photography Portfolio

A static site. No backend, no build tooling beyond one small script.

## Adding a new post

1. Drop the photo into `photos/` (`.jpg`, `.jpeg`, `.png`, `.webp`, or `.gif`).
2. Optional: add a caption by creating a text file with the **same name** as
   the photo, e.g. `photos/golden-gate.jpg` + `photos/golden-gate.txt`
   containing the caption text.
3. Commit and push. If deployed on Vercel (see below), it rebuilds
   `posts.json` automatically — nothing else to run.

Posts are ordered newest-first, using each photo's file date.

## Local preview

`posts.json` is generated by a script, and `fetch()` needs a real server
(not a `file://` URL), so:

```
python3 build.py        # generates/updates posts.json
python3 -m http.server  # serve the folder
```

Then open `http://localhost:8000`.

## Deploying (free)

**Vercel** (recommended):
1. Push this folder to a GitHub repo.
2. Import the repo in Vercel.
3. Framework preset: "Other". `vercel.json` already sets the build command
   (`npm run build`) and output directory, so no manual config needed.
4. Every push rebuilds `posts.json` and redeploys.

**Netlify** works the same way — set build command to `npm run build`,
publish directory to `.`.

## Editing About / Contact

Edit `about.html` directly — it's plain HTML.
